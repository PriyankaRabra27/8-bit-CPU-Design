# Overview
The **Arithmetic Logic Unit (ALU)** is a core component of the CPU.  
It performs arithmetic and logical operations on **8-bit data** based on the control signal provided.

## Inputs and Outputs

| Signal | Width | Description |
|--------|------:|-------------|
| `a`    | 8-bit | Accumulator input |
| `b`    | 8-bit | Immediate operand input |
| `sel`  | 3-bit | ALU operation select signal |
| `y`    | 8-bit | ALU result |
| `z`    | 1-bit | Zero flag (`1` when result is `0`) |

## Operations

| `sel` | Operation |
|------:|-----------|
| `000` | Pass `b` (used for `LOAD`) |
| `001` | `a + b` |
| `010` | `a - b` |
| `011` | `a & b` |
| `100` | `a \| b` |

## Logical Concept
The ALU is implemented using a combinational `case` statement.  
Based on the value of `sel`, it selects one of the supported operations and produces the output `y`.

The zero flag `z` is generated by checking whether the result is equal to zero. This flag is later used by the CPU for conditional branching such as the `JZ` instruction.

## Verilog Concepts Used
- `always @(*)` for combinational logic
- `case(sel)` for operation selection
- continuous assignment for zero flag generation
